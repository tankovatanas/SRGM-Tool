# BBC Надеждност — SRGM Tool

**BBC Надеждност** е Android приложение за анализ на надеждността на софтуер чрез **Software Reliability Growth Models (SRGM)**. Инструментът зарежда исторически данни за грешки, прилага пет математически модела чрез MLE оптимизация и генерира визуални отчети за прогнозиране на бъдещи откази.

---

## Характеристики

- **5 SRGM модела** — едновременно изчисляване и сравнение:
  - Goel-Okumoto (GO)
  - Jelinski-Moranda (JM)
  - Yamada Exponential
  - Delayed S-Shaped
  - Schick-Wolverton (SW)
- **MLE параметрична оценка** чрез Levenberg-Marquardt нелинеен оптимизатор
- **Метрики за качество**: R², AIC, BIC за обективен избор на модел
- **Интерактивна графика** — наблюдавани vs прогнозирани откази
- **Времеви филтър** — slider и текстово поле за прецизен анализ на период
- **Scorecard за сравнение** на всички модели с визуален ранкинг
- **Детайли за параметрите** — α (очакван брой откази), β (скорост на откриване)
- **Автоматизирано резюме** — текстов анализ генериран от модела
- **PDF експорт** — структуриран отчет (`Отчет.pdf`), готов за споделяне
- **Responsive layout** — адаптиран за телефони и таблети (Compact / Medium / Large / XL)
- **Анимиран фон** с математически символи (λ, Σ, ∫, μ, σ, π…)

---

## Поддържани SRGM модели

| Модел | Формула $\mu(t)$ | Параметри |
|-------|-----------------|-----------|
| Goel-Okumoto | $\alpha(1 - e^{-\beta t})$ | $\alpha, \beta$ |
| Jelinski-Moranda | $\alpha(1 - e^{-\beta t})$ | $\alpha, \beta$ |
| Yamada Exponential | $\alpha(1 - (1 + \beta t)e^{-\beta t})$ | $\alpha, \beta$ |
| Delayed S-Shaped | $\alpha(1 - (1 + \beta t)e^{-\beta t})$ | $\alpha, \beta$ |
| Schick-Wolverton | $\alpha(1 - e^{-\beta t^2})$ | $\alpha, \beta$ |

## Формат на входните данни

Приложението поддържа автоматично разпознаване и обработка на два формата на CSV файлове с данни за откази:

### 1. Кумулативни данни (Cumulative Data)
При този формат времето и броят откази са вече натрупани (монотонно нарастващи стойности). Колоните трябва да бъдат `Time` (или `time`) и `Failures` (или `failures`):
```csv
Time,Failures
1.0,2
2.0,5
3.0,9
...
```

### 2. Интервални/Инкрементални данни (Interval Data)
При този формат редовете представляват продължителността на отделните времеви интервали и броя на отказите в тези интервали. Приложението автоматично засича, че данните не са монотонно нарастващи, и ги акумулира хронологично последователно (време: $t_i = \sum_{j=1}^i \Delta t_j$, дефекти: $F_i = \sum_{j=1}^i \Delta F_j$):
```csv
Time,Failures
1.0,2
1.0,3
1.0,4
...
```

*Забележка:* По подразбиране е наличен бутон за директно зареждане на публичния **T39 dataset** от Derek Jones' Reliability-data хранилище, който използва колони `CPU_hour` и `Failures` (интервален формат) и се обработва по интервалната логика.

## Инсталация и стартиране

### Изисквания

- Android Studio Ladybug или по-нова версия
- JDK 11+
- Android устройство или емулатор с API 24+

### Стъпки

```bash
# Клониране на хранилището
git clone https://github.com/tankovatanas/SRGM-Tool.git
cd SRGM-Tool
```

Приложението включва JUnit 4 тестове за валидация на оптимизацията и препроцесирането на данните:

```bash
# Изпълнение на всички unit тестове
./gradlew test
```

## Как се използва

1. **Зареждане на данни** — натиснете "Зареди T39 от GitHub" или качете собствен CSV файл в един от поддържаните формати.
2. **Стартиране на анализ** — натиснете "Стартирай анализ" за едновременно изчисляване на всичките 5 модела.
3. **Сравнение на моделите** — разгледайте scorecard панела с AIC/BIC ранкинг, за да изберете най-подходящия модел.
4. **Филтриране на периода** — използвайте слайдера за интерактивно филтриране и прецизен анализ на времеви прозорец.
5. **Детайлна визуализация** — изберете модел от scorecard-а за проследяване на кривата спрямо реалните точки.
6. **Експортиране на отчет** — натиснете "Експортирай отчет" за генериране и запазване на PDF документ (`Отчет.pdf`).
